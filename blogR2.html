GIT

Git is an open source software which is used to track any changes done in the program.It is basically created for the coordination of work between the team with less effort.

    In the second lecture we focussed on the following softwares:

    • git-Git is a version control tool that will allow you to perform all kinds of operations to fetch data from the central server or push data to it.
    • github-GitHub is a core hosting platform for version control collaboration. GitHub is a company that allows you to host a central repository in a remote server.
    • VS code- Visual Studio Code lets you build and debug modern web and cloud applications.It is an IDE which helps coder to write and test their code.
    • netlify-Netlify is a web developer platform that multiplies productivity. By unifying the elements of the modern decoupled web, from local development to advanced edge logic.

  1. index.html - This file contains the blueprint of website,how it should be made,what it should contain.
  2. Read.me - A README file contains information about the other files in a directory or archive of computer software. A form of documentation, it is usually a simple plain text file called README.
  3. .gitignore - The . gitignore file tells Git which files to ignore when committing your project to the GitHub repository. gitignore is located in the root directory of your repo. / will ignore directories with the name.
  4. adding and removing - In github desktop,when anything is added it shows green color and when anything is removed it shows red color.So,this way you know what changes has been done in github.
  5. commit - A commit, or "revision", is an individual change to a file (or set of files). It's like when you save a file, except with Git, every time you save it creates a unique ID that allows you to keep record of what changes were made when and by who.
  6. push - The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repo.
  7. pull request - Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub. Once a pull request is opened, you can discuss and review the potential changes with collaborators and add follow-up commits before your changes are merged into the base branch.
  8. web hosting - Web hosting is an online service that allows you to publish your website files onto the internet. So, anyone who has access to the internet has access to your website.This is where netlify comes into action
  9. deploys in netlify - There is a section called deploys which contains the history of websites you have deployed from netlify.
  10. branch in github - A branch is a version of the repository that diverges from the main working project. It is a feature available in most modern version control systems.
